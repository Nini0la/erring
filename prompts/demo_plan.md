Demo Scenario

Best hackathon demo:

1. User creates commitment without time.
2. Erring asks for a timeline.
3. User refuses.
4. Erring stores it as untimed.
5. Later Erring retrieves untimed commitments.
6. User gives an update.
7. Erring detects ambiguity and asks whether it relates to an existing project/commitment.
8. Erring updates the correct commitment.
